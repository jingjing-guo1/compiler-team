## 成员数据流转总表

| 成员 | 从谁获取 | 获取的数据（名称 & 内容） | 输出给谁 | 输出的数据（名称 & 内容） |
|------|----------|---------------------------|----------|----------------------------|
| 成员1（集成者） | 成员2,3,4,5 | 接口清单（Token类型、语法树节点类型、三地址码操作码、符号表操作接口） | 全体成员 | `common.h`（公共头文件，包含所有数据结构定义和函数原型） |
| 成员2（词法分析） | 用户/文件 | 源代码字符串（例如 `"int a = 10 + b;"`） | 成员3 | Token列表（Token流），例如 `[KW_INT, ID(a), ASSIGN, NUM(10), PLUS, ID(b), SEMICOLON]` |
| 成员3（语法分析） | 成员2 | Token列表 | 成员4 | 语法树（AST），例如一个根节点为 `AssignStmt` 的树结构 |
| 成员4（语义分析） | 成员3 | 语法树（未带类型） | 成员5 | 带类型标注的语法树 + 符号表（例如每个节点多了一个 `data_type` 字段，以及变量名到类型的映射表） |
| 成员5（中间代码生成） | 成员4 | 带类型的语法树 + 符号表 | 成员6 | 三地址码序列（例如 `t1 = 10`, `t2 = t1 + b`, `a = t2`） |
| 成员6（目标代码生成） | 成员5 | 三地址码序列 | 最终输出 | x86-64汇编代码（例如 `movl $10, %eax` ...） |

## 详细举例说明

### 例子：源代码 `int a = 10 + b;`（假设 b 已声明）

#### 成员2（词法分析）

- **输入**（从源文件读取的字符串）：
  ```
  "int a = 10 + b;"
  ```
- **输出**（Token列表）：
  ```
  [
    { type: TOKEN_KW_INT,   value: NULL },
    { type: TOKEN_IDENTIFIER, value: "a" },
    { type: TOKEN_ASSIGN,   value: NULL },
    { type: TOKEN_INT_LITERAL, value: "10" },
    { type: TOKEN_PLUS,     value: NULL },
    { type: TOKEN_IDENTIFIER, value: "b" },
    { type: TOKEN_SEMICOLON, value: NULL }
  ]
  ```

#### 成员3（语法分析）

- **输入**：上面的 Token 列表
- **输出**（语法树，结构体指针表示）：
  ```
  VarDecl 节点 (name = a, type = int)
    └─ VarDef 节点
         └─ AssignStmt 节点
              ├─ LVal 节点 (name = a)
              └─ BinaryExpr 节点 (op = '+')
                   ├─ Number 节点 (value = 10)
                   └─ LVal 节点 (name = b)
  ```

#### 成员4（语义分析）

- **输入**：上面的语法树（无类型信息）
- **输出**：
  - 相同的语法树结构，但每个表达式节点多了一个 `data_type` 字段（例如 `TYPE_INT`）。
  - 符号表（例如链表）：`{ "b": int }`（变量 `a` 声明时也会加入）。
  - 具体到上面的树，`Number(10)` 的 `data_type = TYPE_INT`，`LVal(b)` 通过查符号表确定类型为 `TYPE_INT`，`BinaryExpr` 的类型也是 `TYPE_INT`。

#### 成员5（中间代码生成）

- **输入**：带类型的语法树 + 符号表
- **输出**（三地址码序列）：
  ```
  1: t1 = 10
  2: t2 = t1 + b
  3: a = t2
  ```
  实际用四元式数组表示：
  ```
  [ (ASSIGN, 10, _, t1),
    (ADD, t1, b, t2),
    (ASSIGN, t2, _, a) ]
  ```

#### 成员6（目标代码生成）

- **输入**：上面的三地址码序列
- **输出**（x86-64 汇编，AT&T 语法）：
  ```asm
  movl $10, %eax
  movl %eax, t1(%rbp)
  movl t1(%rbp), %eax
  addl b(%rbp), %eax
  movl %eax, t2(%rbp)
  movl t2(%rbp), %eax
  movl %eax, a(%rbp)
  ```

## 成员1（集成者）的特殊数据流转

成员1不从上游获取数据流，而是从其他成员获取**接口定义清单**，例如：

- 成员2 提供：`{TOKEN_KW_INT, TOKEN_IDENTIFIER, TOKEN_INT_LITERAL, TOKEN_PLUS, ...}`
- 成员3 提供：`{NODE_VAR_DECL, NODE_ASSIGN_STMT, NODE_IF_STMT, NODE_WHILE_STMT, NODE_BINARY_EXPR, ...}`
- 成员5 提供：`{IR_ADD, IR_SUB, IR_LABEL, IR_GOTO, ...}`
- 成员4 提供：`insert_symbol(name, type)`, `lookup_symbol(name)`

成员1把这些清单统一写入 `common.h`，然后分发给所有人。这样每个人都知道用哪些枚举值、结构体、函数名。

---

## 总结

- **数据是层层转换的**：字符串 → Token → 语法树 → 带类型的语法树 → 三地址码 → 汇编。
- **每个成员只关心自己拿到的输入格式和需要产生的输出格式**，不用了解内部实现。
- 所有数据格式（结构体、枚举）都定义在 `common.h` 中，由成员1统一维护。
