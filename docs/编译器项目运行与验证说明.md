# 编译器项目运行与验证说明

以下描述基于当前项目最终实现版本，用于验证词法分析、语法分析、语义分析、错误处理与多错误收集、中间代码生成以及目标代码生成等模块功能是否正确，同时为课程答辩和 PPT 展示提供参考。

## 1. 拉取最新代码

```bash
git pull origin main
```

正常情况：该命令会从 GitHub 仓库的 main 分支下载团队最新的提交，输出类似“Already up to date.”或显示更新的文件列表。这样能保证使用的是团队最新版本。

## 2. 编译编译器

```bash
"D:\software\mingw64\mingw64\bin\gcc.exe" -Iinclude -o mycc.exe src\*.c
```

正常情况：没有任何错误或警告，并生成 `mycc.exe` 可执行文件。

如果 gcc 已配置到环境变量，也可以直接使用：

```bash
gcc -Iinclude -o mycc.exe src\*.c
```

## 3. 确认编译器已生成

```bash
dir mycc.exe
```

正常情况：列出 `mycc.exe` 文件，文件大小不为 0，且日期为当前编译时间。

## 4. 查看帮助信息

```bash
mycc.exe --help
```

正常情况：输出所有支持的命令行选项及其用途，例如：

```text
Usage: mycc.exe [options] <input.c>

Options:
  -o <file>     Output assembly file (default: output.s)
  -v            Verbose output
  --dump-tokens Print token list
  --dump-ast    Print abstract syntax tree
  --dump-ir     Print intermediate code
  -h, --help    Show this help
```

```text
解释：

-o
指定生成的汇编文件名（默认 output.s）

-v
详细模式，打印编译各阶段信息

--dump-tokens
输出词法分析得到的 Token 序列（验证词法分析模块）

--dump-ast
输出抽象语法树 AST（验证语法分析模块）

--dump-ir
输出中间代码 IR（验证中间代码生成模块）

--help
显示帮助信息
```

## 5. 使用 --dump-tokens 查看词法分析结果

```bash
mycc.exe --dump-tokens test\factorial.c
```

正常情况：逐行输出源代码中的每个词法单元（Token），包括类型和文本值。

例如对于：

```c
int main()
```

可能输出：

```text
KEYWORD_INT    "int"
IDENTIFIER     "main"
LPAREN         "("
RPAREN         ")"
...
```

没有“非法字符”或错误提示。

该功能主要用于验证词法分析模块是否正确工作，并可作为词法分析部分 PPT 展示内容。

## 6. 使用 --dump-ast 查看抽象语法树

```bash
mycc.exe --dump-ast test\factorial.c
```

正常情况：输出树状结构的抽象语法树（AST），清晰地显示函数定义、语句、表达式等程序结构。

例如 factorial 函数的 AST 会包含：

* 函数定义节点
* 参数节点
* 条件判断节点
* 返回语句节点

输出中不应出现：

```text
syntax error
```

该功能主要用于验证语法分析模块是否正确工作，并可作为语法分析部分 PPT 展示内容。

## 7. 使用 --dump-ir 查看中间代码

```bash
mycc.exe --dump-ir test\factorial.c
```

正常情况：输出三地址码（IR）形式的中间表示。

例如：

```text
FUNC_ENTER factorial
  t0 = n == 0
  IF t0 THEN L1
  t1 = n * factorial(n - 1)
  RETURN t1
L1:
  RETURN 1
FUNC_EXIT
```

代码逻辑正确，没有重复块或死循环。

该功能主要用于验证中间代码生成模块是否正确工作，并可作为 IR 模块 PPT 展示内容。

## 8. 正常编译（生成汇编代码）

```bash
mycc.exe test\factorial.c
```

正常情况：

* 不输出任何错误信息
* 成功生成 `output.s` 汇编文件

该文件内容为合法汇编代码，可进一步通过 GCC 链接生成可执行程序。

例如：

```bash
gcc output.s -o factorial.exe
```

运行：

```bash
factorial.exe
```

正常情况：程序输出结果与源程序逻辑一致。

## 9. 验证语义分析与多错误收集模块

```bash
mycc.exe test\semantic_errors.c
```

正常情况：

编译器能够发现程序中的语义问题，并输出对应错误信息。

例如：

* 变量未声明
* 重复定义
* 类型不匹配
* 函数参数错误

同时，编译器不会在发现第一个错误后立即终止，而是继续分析后续代码。

能够一次性报告多个错误信息。

验证内容：

* 错误信息输出
* 错误位置定位
* 多错误收集
* 编译过程稳定性

该部分用于验证语义分析模块以及错误处理与多错误收集模块是否正常工作，也是项目的重要亮点之一。

## 总结

以上所有命令的正常运行结果可用于验证项目各模块实现情况，并为课程答辩和 PPT 展示提供参考依据。

验证内容包括：

* 词法分析模块（Lexer）
* 语法分析模块（Parser）
* 语义分析模块（Semantic）
* 错误处理与多错误收集模块（Error Handling）
* 中间代码生成模块（IR Generation）
* 目标代码生成模块（Code Generation）

当以上验证均通过时，说明编译器已能够完成从源程序到目标汇编代码的完整编译流程。
