# 编译器测试报告

## 1. 测试概述

### 1.1 测试对象

C 子集编译器，源代码文件包括：
- 词法分析器 (`lexer.c`)
- 语法分析器 (`parser.c`)
- 语义分析器 (`semantic.c`)
- 中间代码生成器 (`irgen.c`)
- 目标代码生成器 (`codegen.c`)
- 主控程序 (`main.c`) 及辅助模块 (`utils.c`, `common.h`)

编译器将输入的 `.c` 文件编译为 x86-64 AT&T 汇编，再通过 `gcc` 汇编链接为可执行文件。

### 1.2 测试范围

测试覆盖以下语言特性：

| 类别             | 主要测试点                                         |
| ---------------- | -------------------------------------------------- |
| 基础数值运算     | 整数常量、负数、四则运算、取模、优先级             |
| 变量与赋值       | 变量声明、初始化、赋值、全局变量                   |
| 分支语句         | `if` / `else`，嵌套 if，`else` 悬空匹配，逻辑运算  |
| 循环语句         | `while` 循环，嵌套循环，`break` / `continue`       |
| 混合嵌套         | 循环内嵌套条件，条件内嵌套循环                     |
| 函数调用         | 无参函数、带参函数、函数返回值、`void` 函数        |
| 数组             | 数组声明、元素读写、数组与循环结合                 |
| 常量声明         | `const int` 变量初始化与只读检查                   |
| 错误处理         | 未声明变量、类型不匹配、遗漏分号、作用域违规等     |

### 1.3 测试方法

采用 **自动化脚本测试** + **手动验证测试** 两种方式。

#### 自动化脚本测试（`test_compiler.sh`）

- 编写 20+ 个正向测试（预期编译链接运行成功且返回正确值）和 15+ 个负向测试（预期编译阶段报错）。
- 脚本自动完成：编译编译器 → 对每个测试用例编译源文件 → 链接 → 运行并比对退出码。
- 支持 `--verbose` 和 `--keep` 选项。

#### 手动验证测试

- 基于“手动测试用例校验汇总”文档，运行 14 个典型测试用例，逐条记录运行结果并分析语义正确性。
- 重点关注复杂场景：深层嵌套、短路求值、函数调用等。

## 3. 自动化测试结果

### 3.1 正向测试（预期成功）

共执行 **25** 个正向用例，全部通过（退出码与预期一致）。

部分代表性用例：

| 测试名称               | 源代码片段摘要                                 | 预期返回 | 实际返回 |
| ---------------------- | ---------------------------------------------- | -------- | -------- |
| `basic_assign`         | `int a; a=42; return a;`                       | 42       | 42       |

| `if_true`              | `if(1) a=5; return a;`                         | 5        | 5        |

| `array`                | `int arr[5]; arr[2]=7; return arr[2];`         | 7        | 7        |
| `func_call`            | `int add(int x,int y){return x+y;} main(){return add(3,4);}` | 7 | 7 |

| `const`                | `const int c=10; return c;`                    | 10       | 10       |

> 注意：`short_circuit_and` 测试验证了短路求值，除数为 0 的表达式未被实际求值，程序正常运行。

### 3.2 负向测试（预期编译失败）

共执行 **14** 个负向用例，全部正确触发编译错误且错误信息包含预期关键词。

| 测试名称               | 源代码错误摘要                       | 预期错误关键词   | 结果 |
| ---------------------- | ------------------------------------ | ---------------- | ---- |
| `undeclared_var`       | `return x;`（x 未定义）              | `'x'` 或 `未定义` | 通过 |
| `assign_to_const`      | `const int c=5; c=10;`               | `constant`       | 通过 |
| `break_outside_loop`   | `break;` 不在循环内                  | `break`          | 通过 |
| `void_var`             | `void x;`                            | `void`           | 通过 |
| `func_param_mismatch`  | 函数调用参数个数不符                 | `argument`       | 通过 |
| `missing_return`       | `int f(){}` 无返回值                 | `missing`        | 通过 |
| `array_as_scalar`      | `a=10;` 其中 `a` 是数组              | `index`          | 通过 |
| `redefinition`         | `int x; int x;`                      | `Redefinition`   | 通过 |

所有负向测试均在编译阶段被正确拦截，没有产生错误的目标代码。
