## 三、各模块必须完成的功能

### 成员2（词法分析）—— `src/lexer.c`

- 识别 `common.h` 中定义的所有 `TokenType`，包括关键字、标识符、数字、运算符、界符。
- 正确处理单行注释 `//` 和多行注释 `/* */`。
- 遇到非法字符时，调用 `report_error(COMPILE_ERR_LEXICAL, ...)` 报告错误，然后跳过该字符继续扫描。
- 未闭合的多行注释也要报告错误。
- 正确统计行号（支持 `\n` 和 `\r\n`）。

### 成员3（语法分析）—— `src/parser.c`

- 实现递归下降解析，根据 `Token` 序列构建 AST（节点类型见 `common.h`）。
- 遇到语法错误时，调用 `report_error(COMPILE_ERR_SYNTAX, ...)` 并尝试通过同步点恢复，继续解析后续代码。
- 支持所有语言特性：函数定义、变量声明、`if`、`while`、`return`、`break`、`continue`、复合语句、各种表达式（含运算符优先级）。
- 删除未使用的函数声明，避免编译警告。

### 成员4（语义分析）—— `src/semantic.c`

- 实现符号表（作用域管理、符号插入与查找）。
- 检查变量是否已声明、函数是否已定义、变量是否重复定义。
- 检查类型匹配：赋值左右类型、函数参数类型、返回值类型。
- 检查 `break` / `continue` 是否出现在循环内部。
- 所有语义错误通过 `report_error(COMPILE_ERR_SEMANTIC, ...)` 报告，并继续分析其他部分。
- 遍历 AST 后，为每个表达式节点标注 `data_type`。

### 成员5（中间代码生成）—— `src/irgen.c`

- 遍历 AST，生成三地址码（四元式序列），操作码见 `common.h` 中的 `IROp`。
- 为每个表达式引入临时变量（`new_temp()`），正确翻译算术、关系、逻辑运算。
- 为 `if`、`while` 生成标签和条件/无条件跳转。
- 处理 `break` / `continue` 的跳转标签。
- 为函数调用生成 `IR_CALL`，并处理参数传递（至少预留接口）。
- 确保生成的四元式可以被 `codegen.c` 正确翻译。

### 成员6（目标代码生成与测试）—— `src/codegen.c`

- 将四元式序列翻译为 x86-64 汇编（AT&T 语法）。
- 处理所有 `IROp` 对应的汇编指令，包括函数进入/退出、调用、返回值。
- 实现简单的栈帧管理（局部变量和临时变量分配到栈上）。
- 编写测试用例（放在 `test/` 目录），验证编译器的正确性。
- 测试用例尽量全一些，可以覆盖到一些错误场景之类的。
- 测试时，若发现其他模块的 bug，及时反馈给对应成员。
- 生成可执行文件并运行，确保结果正确。

## 四、提交代码流程

1. 本地编译通过：`make clean && make`
2. 提交并推送到自己的远程分支：
   ```bash
   git add src/你的文件.c
   git commit -m "feat(模块名): 实现XXX功能"
   git push origin feature/你的模块名
   ```
3. 在 GitHub 上发起 Pull Request，目标分支 `main`，等待我（成员1）审查。
4. 根据审查意见修改，推送后 PR 自动更新。
5. 审查通过后由我合并。

---

## 五、重要规则

- ❌ **禁止直接推送 `main` 分支**（受保护）。
- ❌ **禁止私自修改 `common.h`、`main.c`、`utils.c`、`Makefile`**。如需修改请先沟通。
- ✅ **错误处理**：所有可恢复的错误必须调用 `report_error()`，不能 `exit`。
- ✅ **多错误收集**：即使遇到错误也要尽量继续编译，直到错误数达到上限（`max_errors = 20`）。
- ✅ **内存管理**：使用 `safe_malloc` / `safe_realloc` 分配；在合适的地方释放（使用已有的 `free_ast`、`free_ir` 等）。
- ✅ **每次工作前**：`git checkout main && git pull`，再合并到自己的分支。
