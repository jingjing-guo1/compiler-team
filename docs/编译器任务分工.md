## 二、每个成员的详细任务说明

### 👤 成员1：项目集成者

**你的核心目标**：  
确保整个项目能够从“源代码文件”走到“汇编代码文件”。你不具体实现词法、语法等算法，但你要负责定义大家怎么对接，并且把所有人的代码组装成一个可以运行的程序。

**你具体要做的事情（按步骤）：**

1. **定义公共数据结构和接口**（这是最重要的任务）  
   - 根据其他成员提交的接口清单，统一编写 `common.h`，包含：  
     - Token类型枚举、Token结构体  
     - 语法树节点类型枚举、语法树节点结构体（预留类型字段）  
     - 三地址码操作码枚举、四元式结构体  
     - **所有模块的公开函数原型**（函数名、参数类型、返回值，由你预先定义并通知各成员）  
   - 将 `common.h` 提交到Git仓库的 `include/` 目录，并通知所有人使用。  

2. **搭建主控程序**  
   - 写 `main.c`，流程如下：  
     1. 读取源文件（如 `test.c`）到字符串。  
     2. 调用成员2的 `lex_analyze()` 得到Token数组。  
     3. 调用成员3的 `parse()` 得到语法树根节点。  
     4. 调用成员4的 `semantic_analyze()` 得到带注释的树和符号表。  
     5. 调用成员5的 `gen_ir()` 得到三地址码序列。  
     6. 调用成员6的 `gen_assembly()` 得到汇编字符串。  
     7. 写入输出文件（如 `output.s`）。  

3. **组织版本管理**  
   - 创建Git仓库（GitHub/Gitee），添加所有成员。  
   - 建立目录：`src/`（每人一个`.c`文件）、`include/`（公共头文件）、`test/`（测试用例）。  
   - 写好 `.gitignore` 和 `Makefile`。  

4. **负责编译和运行脚本**  
   - 写一个 `Makefile`，能够一次性编译所有模块并生成可执行文件 `compiler`。  

**与他人协作的要点**：  
- 收集每个成员提交的接口定义清单（见下文各成员任务），及时更新 `common.h`。  
- 提醒所有成员从Git仓库拉取最新的 `common.h` 并基于它开发。  
- 不负责调试具体模块的错误，只负责接口统一和集成脚本。  

**你需要从其他成员处获取的内容：**  
- 成员2：Token类型清单（需要哪些Token种类）  
- 成员3：语法树节点类型清单（需要哪些节点种类）  
- 成员5：三地址码操作码清单（需要哪些指令）  
- 成员4：符号表的基本操作接口（如 `insert_symbol`, `lookup_symbol` 等，不需要完整实现，只需接口说明）  
- 所有成员：如有对函数原型的特殊需求，可提前与成员1协商，由成员1统一写在 `common.h` 中  

**答辩PPT内容**：  
- 整体架构图。  
- 公共数据结构定义。  
- 主控程序流程图。  

**答辩时你讲的部分**：  
- 项目背景、总体设计。  
- 如何组织模块接口和集成。  
- 展示最终运行成功的一个例子。

### 👤 成员2：词法分析负责人

**你的核心目标**：  
将源代码文本（一个字符串）变成一个个的单词（Token），例如 `int a = 10;` → `[TOKEN_KW_INT]`, `[TOKEN_IDENTIFIER(a)]`, `[TOKEN_ASSIGN]`, `[TOKEN_INT_LITERAL(10)]`, `[TOKEN_SEMICOLON]`。

**你需要提供给成员1的内容（在编码前提交）：**  
- 一份Token类型清单，例如：  
  - 关键字：`int`, `void`, `if`, `else`, `while`, `return`, `const`, `break`, `continue`  
  - 标识符  
  - 整数常量  
  - 运算符：`+`, `-`, `*`, `/`, `%`, `=`, `==`, `!=`, `<`, `<=`, `>`, `>=`, `!`, `&&`, `||`  
  - 界符：`;`, `,`, `(`, `)`, `{`, `}`, `[`, `]`  
  - 注释（可选）  

**你具体要做的事情：**  
1. 使用成员1定义的 `TokenType` 枚举和 `Token` 结构体（来自 `common.h`）。  
2. 实现词法分析函数，函数原型由成员1在 `common.h` 中预先声明（例如 `TokenList* lex_analyze(const char* source_code);`）。  
   - 扫描字符，识别单词，填充 `Token` 结构体。  
   - 跳过空格、换行、注释（`//` 和 `/* */`）。  
3. 提供多错误收集：遇到非法字符时调用 `report_error()` 报告错误，并跳过该字符继续扫描；未闭合的注释也要报告错误。  

**输入输出示例：**  
- 输入：`"int a = 10 + b;"`  
- 输出：`[TOKEN_KW_INT] [TOKEN_IDENTIFIER(a)] [TOKEN_ASSIGN] [TOKEN_INT_LITERAL(10)] [TOKEN_PLUS] [TOKEN_IDENTIFIER(b)] [TOKEN_SEMICOLON]`  

**与其他成员的关联：**  
- 你的输出 `TokenList` 是成员3的输入。  
- 你需要使用成员1提供的 `common.h`。  

**答辩PPT内容**：  
- 词法分析的作用。  
- 状态转换图示例。  
- 核心代码片段。  
- 展示源代码如何变成Token序列。  

**答辩时你讲的部分**：  
- 如何把字符串切成单词。  
- 如何处理不同类型单词。  
- 举例演示。

### 👤 成员3：语法分析负责人

**你的核心目标**：  
按照语言的语法规则，检查Token序列的结构是否正确，并构建出一棵语法树（AST）。例如 `a = 10 + b;` 应该变成一个“赋值语句节点”，下面有“变量节点”、“表达式节点”等。

**你需要提供给成员1的内容（在编码前提交）：**  
- 语法树节点类型清单，例如：  
  - `NODE_PROG`（根节点）  
  - `NODE_VAR_DECL`, `NODE_VAR_DEF`（变量声明与定义）  
  - `NODE_FUNC_DEF`, `NODE_FUNC_FPARAM`（函数定义与参数）  
  - `NODE_ASSIGN_STMT`（赋值语句）  
  - `NODE_IF_STMT`, `NODE_WHILE_STMT`（控制流）  
  - `NODE_RETURN_STMT`, `NODE_BREAK_STMT`, `NODE_CONTINUE_STMT`  
  - `NODE_BLOCK`（复合语句）  
  - `NODE_BINARY_EXPR`, `NODE_UNARY_EXPR`, `NODE_FUNC_CALL`, `NODE_LVAL`, `NODE_NUMBER`（表达式）  

**你具体要做的事情：**  
1. 使用成员1定义的语法树节点结构体（来自 `common.h`）。  
2. 采用递归下降分析法：为每个语法成分写解析函数，函数原型由成员1在 `common.h` 中预先声明（例如 `ASTNode* parse(TokenList* tokens);`）。  
3. 同步Token指针，匹配并构建节点。  
4. 多错误处理：遇到语法错误时调用 `report_error()`，并通过同步点恢复（跳过当前语句），继续解析后续代码。  

**输入输出示例：**  
- 输入Token流：`[TOKEN_IDENTIFIER(a)] [TOKEN_ASSIGN] [TOKEN_INT_LITERAL(10)] [TOKEN_PLUS] [TOKEN_IDENTIFIER(b)] [TOKEN_SEMICOLON]`  
- 输出语法树：  
  ```
  AssignStmt
    left: LVal(a)
    right: BinaryExpr(+)
            left: Number(10)
            right: LVal(b)
  ```

**与其他成员的关联：**  
- 上游（成员2）：需要成员2输出的TokenList。  
- 下游（成员4）：将语法树根节点传给成员4。  

**答辩PPT内容**：  
- 上下文无关文法简介。  
- 递归下降分析过程。  
- 语法树示例图。  
- 核心解析函数代码片段。  

**答辩时你讲的部分**：  
- 如何利用Token序列建立树结构。  
- 如何处理运算符优先级。  
- 展示一个程序片段对应的语法树。

### 👤 成员4：语义分析负责人

**你的核心目标**：  
检查程序的“合理性”：变量是否定义、类型是否匹配、函数调用是否正确等。同时构建符号表，并将类型信息挂到语法树节点上。

**你需要提供给成员1的内容（在编码前提交）：**  
- 符号表的基本设计思路（不需要写全部代码，只需说明你需要的符号表操作接口），例如：  
  - `void init_symbol_table(SymbolTable *table);`  
  - `void enter_scope(SymbolTable *table);`  
  - `void exit_scope(SymbolTable *table);`  
  - `int insert_symbol(SymbolTable *table, const char *name, DataType type, SymbolKind kind, int line);`  
  - `Symbol *lookup_symbol(SymbolTable *table, const char *name);`  

**你具体要做的事情：**  
1. 使用成员1定义的数据类型枚举（`DataType`：`TYPE_INT`, `TYPE_VOID` 等）。  
2. 实现符号表（链表或哈希表），支持作用域嵌套。  
3. 遍历语法树：  
   - 变量声明时插入符号表。  
   - 变量使用时查找符号表，未定义则报错。  
   - 类型检查（例如 `int + int` 合法，`void` 不能用于运算）。  
   - 将表达式的类型挂到语法树节点的 `data_type` 字段。  
4. 额外检查：函数参数个数与类型、`break`/`continue` 是否在循环内。  
5. 多错误收集：遇到语义错误时调用 `report_error()` 并继续分析其他部分。  
6. 语义分析函数的原型由成员1在 `common.h` 中预先声明（例如 `int semantic_analyze(ASTNode *root, SymbolTable *table);`）。  

**输入输出示例：**  
- 输入语法树：`AssignStmt(left: LVal(a), right: Number(10))`，且之前有 `int a;` 声明。  
- 输出：符号表记录 `a: TYPE_INT`，语法树节点上 `Number(10)` 的 `data_type = TYPE_INT`。  

**与其他成员的关联：**  
- 上游（成员3）：需要成员3提供的语法树结构。  
- 下游（成员5）：将带类型标注的语法树和符号表传递给成员5。  

**答辩PPT内容**：  
- 符号表的作用和组织方式。  
- 类型检查规则。  
- 语义错误示例及报错信息。  
- 代码片段：如何遍历树并填充类型。  

**答辩时你讲的部分**：  
- 为什么要做语义分析？  
- 如何管理符号表？  
- 如何发现变量未定义或类型不匹配的错误。

### 👤 成员5：中间代码生成负责人

**你的核心目标**：  
将语法树翻译成三地址码（Three-Address Code），例如 `t1 = a + b`，`t2 = t1 * 2` 等，简化目标代码生成。

**你需要提供给成员1的内容（在编码前提交）：**  
- 三地址码操作码清单，例如：  
  - 算术：`IR_ADD`, `IR_SUB`, `IR_MUL`, `IR_DIV`, `IR_MOD`  
  - 关系与逻辑：`IR_LT`, `IR_LE`, `IR_GT`, `IR_GE`, `IR_EQ`, `IR_NE`, `IR_AND`, `IR_OR`, `IR_NOT`  
  - 控制流：`IR_LABEL`, `IR_GOTO`, `IR_IF_FALSE_GOTO`  
  - 函数：`IR_RETURN`, `IR_CALL`, `IR_FUNC_ENTER`, `IR_FUNC_EXIT`  
  - 赋值：`IR_ASSIGN`  

**你具体要做的事情：**  
1. 使用成员1定义的四元式结构体（来自 `common.h`）。  
2. 遍历语法树，按规则生成代码，函数原型由成员1在 `common.h` 中预先声明（例如 `IRCode* gen_ir(ASTNode* root);`）。  
   - 表达式：递归生成左右子树的代码，产生临时变量（`new_temp()`）。  
   - 赋值语句：生成右部表达式代码，然后生成 `IR_ASSIGN`。  
   - if语句：生成条件代码、条件跳转、then块、else块、结束标签。  
   - while语句：生成循环标签、条件代码、条件跳转、循环体、跳回。  
   - 函数：生成函数进入/退出标签，处理参数与返回值。  
3. 使用临时变量计数器 `new_temp()`。  
4. 多错误处理：生成过程遇到错误时调用 `report_error()`，可提前停止（因为无错误才调用该模块）。  

**输入输出示例：**  
- 输入语法树：`AssignStmt(left: LVal(a), right: BinaryExpr(+, left: Number(10), right: LVal(b)))`  
- 输出三地址码：  
  ```
  t1 = 10
  t2 = t1 + b
  a = t2
  ```

**与其他成员的关联：**  
- 上游（成员4）：需要成员4提供的带类型语法树（实际生成代码时主要看结构）。  
- 下游（成员6）：将产生的三地址码列表传给成员6。  

**答辩PPT内容**：  
- 什么是三地址码？为什么需要中间代码？  
- 如何翻译表达式（临时变量）。  
- 如何翻译if和while（标签和跳转）。  
- 完整示例。  

**答辩时你讲的部分**：  
- 展示如何把树形结构转化为线性指令序列。  
- 解释临时变量的作用。  
- 对比直接生成汇编的困难。

### 👤 成员6：目标代码生成与测试负责人

**你的核心目标**：  
将三地址码翻译成x86-64汇编代码，并设计测试用例验证整个编译器的正确性。

**你需要提供给成员1的内容（在编码前提交）：**  
- 无特殊清单，但可以与成员1协商函数原型（例如 `void gen_assembly(IRCode* ir, const char* output_filename);`）。  

**你具体要做的事情：**  
1. 实现简单的寄存器分配（或全部变量放内存，使用栈帧）。  
2. 逐条翻译三地址码到x86-64汇编（AT&T 语法），函数原型由成员1在 `common.h` 中预先声明，你需严格按照该原型实现。  
   - `IR_ADD t1, t2, t3` → `movl t1(%rbp), %eax`; `addl t2(%rbp), %eax`; `movl %eax, t3(%rbp)`  
   - `IR_IF_FALSE_GOTO t1, L1` → `cmpl $0, t1(%rbp)`; `je L1`  
   - `IR_CALL func` → `call func`  
3. 生成完整的汇编文件（段定义、标签、指令）。  
4. 编写测试用例（放在 `test/` 目录下），用编译器生成汇编，再用 `gcc -no-pie` 链接运行，验证正确性。  
5. 整理测试报告，反馈发现的其他模块 bug。  
6. 多错误处理：若遇到不支持的操作或严重错误，调用 `report_error()`。  

**与其他成员的关联：**  
- 上游（成员5）：依赖成员5提供的三地址码格式。  
- 整体测试：发现错误时反馈给对应成员修改。  

**答辩PPT内容**：  
- 目标代码生成任务。

**答辩时你讲的部分**：

## 三、协作提醒

- **Git使用**：成员1会创建仓库，所有人 `clone` 后，在各自分支上工作。每次成员1更新 `common.h` 后，在群内提醒，其他人必须 `pull` 最新版本。

- **接口变更**：任何人需要修改 `common.h` 中已定义的内容（包括数据结构、枚举、函数原型等），必须先向成员1提出，由成员1评估后统一修改并通知所有人。禁止私自修改本地的 `common.h`。

- **函数原型调整**：成员1会在 `common.h` 中给出初版函数原型。如果在编码过程中发现原型不合理（如参数不足、返回值不合适），该成员应立即在群里提出修改建议并说明原因。成员1评估后决定是否修改，若修改则更新 `common.h` 并推送，同时通知所有人 `git pull` 同步。所有成员需及时拉取最新版本并调整自己的代码。

- **集成测试**：成员6负责最终集成测试，但所有成员在开发过程中应及时拉取Git最新代码。遇到集成问题，先检查自己的模块是否与最新接口匹配，再联系相关成员协商解决。
