# C语言子集编译器

## 一、项目简介

本项目为编译原理课程大作业，实现了一个面向 C 语言子集的编译器。编译器采用经典的多阶段结构设计，包括词法分析、语法分析、语义分析、中间代码生成以及目标代码生成等模块，能够将输入的 C 语言源程序转换为目标汇编代码。

项目采用模块化设计，各阶段相互独立，通过统一的数据结构进行信息传递，便于开发、测试和维护。

---

## 二、实现功能

### 1. 词法分析（Lexer）

实现源程序扫描与 Token 生成，支持：

* 关键字识别
* 标识符识别
* 整数字面量识别
* 运算符识别
* 分隔符识别
* 单行注释与多行注释处理
* 非法字符检测

---

### 2. 语法分析（Parser）

采用递归下降分析方法构建抽象语法树（AST）。

支持：

* 变量声明
* 常量声明
* 函数定义
* 复合语句
* if / else
* while
* return
* break
* continue
* 表达式求值
* 函数调用

---

### 3. 语义分析（Semantic Analysis）

实现符号表管理与语义检查。

支持：

* 变量重复定义检查
* 未声明变量检查
* 函数定义检查
* 函数参数检查
* 返回值检查
* 类型匹配检查
* break / continue 合法性检查

---

### 4. 中间代码生成（IR Generation）

将抽象语法树转换为三地址码（IR）。

支持：

* 算术运算
* 赋值语句
* 条件跳转
* 循环结构
* 函数调用
* 返回语句

---

### 5. 目标代码生成（Code Generation）

根据中间代码生成目标汇编代码。

支持：

* 算术运算指令生成
* 条件跳转指令生成
* 函数调用与返回
* 基本栈帧管理
* 汇编文件输出

---

### 6. 错误处理与多错误收集模块（Error Handling）

项目设计了独立的错误处理模块，负责统一管理编译过程中产生的错误信息。

主要功能：

* 统一错误报告与记录
* 多错误收集
* 错误恢复与继续分析
* 错误数量统计

当编译过程中发现错误时，编译器不会立即终止，而是尽可能继续完成后续分析，并在编译结束后统一输出错误信息。

例如，对于同一源程序中存在的多个问题，编译器能够一次性给出多条错误提示，而不是仅报告第一个错误后退出，从而提高调试效率和编译器的实用性。

该模块增强了编译器的健壮性，也是项目的重要组成部分。

## 三、编译流程
```text
源程序

↓

词法分析（Lexer）

↓

Token序列

↓

语法分析（Parser）

↓

抽象语法树（AST）

↓

语义分析（Semantic）

↓

符号表检查

↓

中间代码生成（IR）

↓

三地址码

↓

目标代码生成（CodeGen）

↓

汇编代码（.s）
```

## 五、编译与运行

### 编译项目

```bash
gcc -Iinclude -o mycc.exe src/*.c
```

或使用：

```bash
make
```

---

### 编译测试程序

```bash
mycc.exe test/factorial.c
```

编译成功后生成：

```text
output.s
```

---

### 汇编链接

```bash
gcc output.s -o factorial.exe
```

运行：

```bash
factorial.exe
```
